# Script Writing

YouTube 영상 스크립트를 작성하는 스킬. 시청자를 첫 30초에 잡고 끝까지 유지하는 대본을 만든다.

## 워크플로우

### 1. 입력 분석

다음 정보를 확인한다:
- **주제 및 트렌드 데이터**: 트렌드 보고서 또는 사용자 지정 주제
- **채널 톤**: 교육적, 유머러스, 진지함, 캐주얼 등
- **타겟 영상 길이**: 기본 10~15분 (약 1,500~2,200단어)
- **타겟 오디언스**: 기술 수준, 관심사

트렌드 보고서가 `_workspace/01_researcher_trend_report.md`에 있으면 읽어서 활용한다.

### 2. 구조 설계

영상을 다음 구조로 설계한다:

**후킹 (0:00~0:30)**
- 시청자의 핵심 고민이나 호기심을 건드리는 오프닝
- "이 영상을 끝까지 보면 ~를 알게 됩니다" 식의 가치 제안
- 충격적 사실, 반직관적 주장, 공감할 수 있는 상황 중 하나 활용

**본문 (섹션별)**
- 각 섹션 시작에 "다음이 궁금하게 만드는" 전환 문구
- 추상적 설명 → 구체적 예시/사례 패턴
- 3~5개 핵심 포인트로 구성 (너무 많으면 산만)
- 중간중간 시청자 참여 유도 (질문, 상상 요청)

**CTA (마지막 30초)**
- 핵심 내용 1문장 요약
- 구독/좋아요 요청 (자연스러운 맥락에서)
- 다음 영상 예고 또는 관련 영상 추천

### 3. 스크립트 작성

- **구어체**를 사용한다. 문어체로 쓰면 읽을 때 부자연스럽다
- 한 문장은 짧게. 복잡한 종속절을 피한다
- 전문 용어는 바로 다음에 쉬운 설명을 붙인다
- "[B-roll: ...]", "[화면: ...]" 같은 영상 연출 노트를 포함한다
- 강조할 부분은 **볼드**로 표시한다

#### 출처/크레딧 처리 규칙

- **기본 원칙**: 참고자료의 **원문 작성자나 출처를 영상 본문에서 언급하지 않는 것을 기본값**으로 한다. 낭독 대본에 "○○님 블로그", "○○ 기사에 따르면" 같은 크레딧 문장을 넣지 않는다.
- **이유**: 시청자 몰입을 방해하고 영상 톤을 정보성 요약에서 "남의 글 소개"로 바꾼다. 검색 노출 관점에서도 크레딧 문장은 키워드 밀도를 희석시킨다.
- **영상 외 노출**: 출처·원문 링크는 영상 설명란(Description)과 핑거드 코멘트에 배치한다. SEO 에이전트가 설명문에 참고자료 섹션을 포함하도록 한다.
- **예외 (이때만 영상 내 언급)**:
  - 사용자가 명시적으로 "출처를 밝혀달라"고 요청한 경우
  - 특정 연구·논문·통계의 신뢰성을 영상 내에서 담보해야 하는 경우 (예: "스탠포드 2025 연구에서…")
  - 인터뷰/협업 콘텐츠처럼 크레딧 자체가 콘텐츠의 일부인 경우
- 예외에 해당하지 않으면 후킹에도 원문 크레딧을 넣지 않는다.

### 4. 산출물 저장

`script/02_writer_script.md`에 다음 형식으로 저장한다 (`script/` 디렉토리가 없으면 생성):

```markdown
# 영상 스크립트: [제목]

## 메타 정보
- 예상 분량: [X분 Y초]
- 톤: [교육적/유머/진지...]
- 타겟: [오디언스 설명]

## 후킹 (0:00~0:30)
[스크립트 내용]

## 본문

### 섹션 1: [소제목] (0:30~3:00)
[스크립트 내용]
[B-roll: 설명]

### 섹션 2: [소제목] (3:00~6:00)
...

## CTA (X:XX~끝)
[스크립트 내용]

## 핵심 키워드
- [SEO 에이전트 참고용 키워드 목록]
```

## 원칙

- 시청 유지율이 최우선. 모든 구성 요소가 "다음이 궁금하게" 만드는지 자문한다
- 채널 톤에 맞추되, 기본은 친근하고 에너지 있는 구어체
- 분량을 지킨다. 10분 영상 스크립트가 30분 분량이면 실패
- 핵심 키워드를 스크립트에 자연스럽게 녹여 SEO에 기여한다
